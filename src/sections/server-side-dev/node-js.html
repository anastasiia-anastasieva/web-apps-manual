Node.JS

Node.js є потужною крос-платформною середовищем виконання, яка дозволяє розробникам використовувати JavaScript для написання серверної логіки. Це робить Node.js ідеальним вибором для створення швидких і масштабованих мережевих додатків.

Асинхронне Виконання

Однією з основних особливостей Node.js є його асинхронна, подієва модель виконання, що дозволяє ефективно обробляти велику кількість паралельних з'єднань без блокування потоку виконання. Це означає, що операції вводу/виводу, такі як доступ до файлової системи, мережеві запити або звернення до баз даних, виконуються у фоновому режимі, дозволяючи основному потоку продовжувати роботу без очікування завершення цих операцій.

Модулі

Node.js використовує модульну архітектуру, що сприяє організації коду та його повторному використанню. Кожен файл у Node.js може бути модулем, який експортує об'єкти, функції або змінні, які можуть бути імпортовані та використані в інших файлах за допомогою require() функції. Це дозволяє розбивати додаток на менші, керовані частини та сприяє співпраці та розподілу коду.

Роутинг

Роутинг в Node.js відповідає за визначення, як додаток реагує на клієнтський запит до конкретного кінцевого пункту, який є URL (або шляхом) та конкретним HTTP запитом (GET, POST, і т.д.). Роутинг дозволяє розробникам легко керувати різними запитами клієнтів, направляючи їх на відповідні обробники запитів.

Middleware

Middleware у Node.js — це функції, які мають доступ до об'єкта запиту (req), об'єкта відповіді (res), та наступної функції в циклі запиту-відповіді додатку. Вони використовуються для виконання коду, зміни об'єктів запиту та відповіді, завершення циклу запиту-відповіді або виклику наступної функції у стеку. Middleware можуть виконувати багато завдань, таких як логування, парсинг тіла запиту, аутентифікація користувачів тощо.

Приклад простого веб-сервера на Node.js

            const http = require('http');

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello, World!\n');
});

const PORT = 3000;
server.listen(PORT, () => {
  console.log(`Server running at http://localhost:${PORT}/`);
});

Цей базовий приклад створює веб-сервер, який слухає на порту 3000 та відповідає "Hello, World!" на всі запити.

Node.js надає потужний набір інструментів для розробки масштабованих веб-додатків, включаючи підтримку асинхронного програмування, модульну структуру, гнучке управління маршрутами та зручну роботу з middleware. Використання цих можливостей дозволяє створювати ефективні та високопродуктивні веб-додатки.

Практичне завдання

Розробіть REST API, використовуючи Node.js та фреймворк Express, яке дозволить виконувати основні операції CRUD (Створення, Читання, Оновлення, Видалення) для користувачів. Дані користувачів можуть зберігатися у пам'яті сервера (для спрощення, без використання бази даних).

Кроки виконання:

Крок 1: Ініціалізація проекту:

  • Створіть нову папку для вашого проекту та ініціалізуйте новий Node.js проект за допомогою команди npm init.

  • Встановіть Express як залежність проекту за допомогою npm install express.

Крок 2: Створення основного сервера:

  • У корені проекту створіть файл server.js.

  • Імпортуйте Express та створіть новий екземпляр додатку.

  • Налаштуйте додаток на прослуховування HTTP запитів на певному порті (наприклад, 3000).

Крок 3: Реалізація CRUD операцій:

  • Створіть модель User з базовими полями, такими як id, name, та email.

  • Реалізуйте ендпойнти для кожної з CRUD операцій:
    POST /users для створення нового користувача.
    GET /users для отримання списку всіх користувачів.
    GET /users/:id для отримання детальної інформації про конкретного користувача.
    PUT /users/:id для оновлення даних користувача.
    DELETE /users/:id для видалення користувача.

Крок 4: Використання Middleware:

  • Додайте middleware, яке логує деталі кожного запиту (наприклад, HTTP метод та шлях) до консолі перед обробкою запиту.

  • Реалізуйте просту перевірку автентифікації, яка перевіряє наявність певного заголовка у запиті, перед дозволом доступу до деяких ендпойнтів.

Крок 5: Тестування API:

  • Використовуйте Postman або будь-який інший HTTP клієнт для тестування реалізованих ендпойнтів.

Це завдання допоможе вам зрозуміти основи роботи з Node.js, Express, а також навчити вас основам створення REST API та роботі з асинхронним програмуванням.
